# 平台能力使用原则

文档搭子优先复用办公平台能力，而不是重复做一套聊天、文件和协作系统。

## 飞书优先使用

- 机器人消息：让用户在群里 @ 文档搭子。
- 事件订阅：接收用户消息、文件消息和群聊上下文。
- 消息卡片：用于冲突确认、整理完成通知、待办提醒。
- 云文档/知识库：作为项目记忆的默认可见存放地。
- 云空间/文件能力：保存或读取用户上传的资料。
- 组织身份：用飞书用户和群聊身份做权限边界。

## 钉钉优先使用

- 企业内部机器人或应用机器人。
- 互动卡片。
- 钉钉文档和钉盘。
- 企业组织身份和群聊上下文。

## 企业微信优先使用

- 企业应用消息回调。
- 群聊和客户群场景。
- 图文/卡片消息。
- 微盘或企业已有文档系统。
- 企业成员身份和客户联系上下文。

## 我们自己只做什么

- 项目识别。
- 内容抽取和整理。
- 项目记忆维护。
- 冲突检测。
- 问答和汇报生成。
- 平台之间统一的 adapter 协议。

这样普通用户不需要学习一个新系统，管理员也不用迁移现有文档体系。
